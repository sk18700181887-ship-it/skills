# 分维度对策模板

## R — 原文 (Reading)

> 引进层面：优化人才结构、降低准入门槛、扩大引才范围、拓宽引才渠道。培育层面：吸纳能人入党、专业化人才。留用层面：人性化管理、实施正面激励、营造包容的工作氛围、优化人才选拔机制。
> 生态保护三层次：生态补偿（对受损者经济补偿）→ 退耕还湿/保护性耕作（修复生态）→ 保护性开发/平衡保护与开发（预防进一步破坏）。
> 产业竞争力三步：统一生产标准（质量一致）→ 销售渠道拓宽（打通线上线下）→ 提升产品认可度（品牌化、认证、溯源）。
>
> — 申论规范词积累课程，人才队伍建设/生态建设/产业发展模块

---

## I — 方法论骨架 (Interpretation)

每个政策领域的对策都有固定的维度结构——这不是"多角度分析"的空话，而是5套具体可套用的维度模板。

**人才领域** → 引-育-留三维：引进（降门槛、扩范围、宽渠道、分层引）→ 培育（专业化、吸纳能人）→ 留用（人性化管理、正面激励、包容氛围、科学选拔）。

**生态领域** → 补偿-修复-预防三段：补偿（对受损者经济补偿）→ 修复（退耕还湿、保护性耕作、无害化处理）→ 预防（保护性开发、平衡保护与开发）。

**产业领域** → 标准-渠道-品牌三步：统一标准（保证质量一致性）→ 拓宽渠道（打通线上线下销售）→ 提升认可度（品牌化、认证、溯源）。

**民生就业领域** → 多元-就近-匹配三维：多元化（拓宽就业途径）→ 就近化（家门口就业）→ 匹配化（解决技能与岗位错配）。

**普法领域** → 通俗-创新-丰富三维：通俗性（老百姓听得懂）→ 创新性（方式新颖，如顺口溜、短视频）→ 丰富性（内容全面，既讲禁止也讲维权）。

模板的核心作用：答完对策后逐维度检查，确保答案不遗漏关键维度。一个题可能涉及多个领域的模板，需要组合使用。

## A1 — 书中的应用 (Past Application)

### 案例 1: Q市三层次人才引育体系（人才：引-育-留模板）
- **问题**: 真题要求概括Q市人才引进工作做法
- **方法论的使用**: 材料描述Q市"不仅要引进突破卡脖子技术难题的学术人才，也需要培育储备大量技能人才，更要广泛建设基础性人才队伍"——完整覆盖"引-育-留"三维：引进层面有降低准入门槛（取消博士年龄限制、放宽社保年限）、扩大引才范围（普通院校技校生也纳入）、拓宽引才渠道（第三方合作、双向挂职、项目合作等）；留用层面有人性化管理（以人为本）、正面激励（肯定点滴进步）、包容氛围（允许犯错）、优化选拔机制（公开竞聘取代上级任命）
- **结论**: 答案按引-育-留三个维度组织，确保每个维度都有对应规范词
- **结果**: 规范答案覆盖：优化人才结构、降低人才准入门槛、扩大引才范围、引进高端人才、引进基础性人才、拓宽引才渠道、人性化管理、实施正面激励、营造包容的工作氛围、优化人才选拔机制

### 案例 2: T村葡萄产业与L县溯源体系（产业：标准-渠道-品牌模板）
- **问题**: 真题要求概括农技员帮助T村发展葡萄产业的做法及L县智慧农业经验
- **方法论的使用**: T村案例中，农技员小李召集专家制定《T村葡萄产品生产标准》，实现从选种到栽苗、生长、采摘的"四统一"——对应"统一生产标准"维度；阳山水蜜桃入驻美团、淘宝、京东等电商平台——对应"销售渠道拓宽"维度；W县马铃薯设计商标logo、争取国家有机食品认证——对应"提升产品认可度"维度；L县建立农产品电子二维码溯源机制——同样属于品牌维度
- **结论**: 产业类对策按"标准→渠道→品牌"路径组织，三个维度缺一不可
- **结果**: 规范答案覆盖：统一生产标准、销售渠道拓宽、建立溯源机制、提升产品认可度

### 案例 3: 盐城湿地保护与黑土地保护（生态：补偿-修复-预防模板）
- **问题**: 真题要求概括湿地保护和黑土地保护的举措
- **方法论的使用**: 盐城案例中，农户因野鸟毁稻获经济补偿——对应"生态补偿"（补偿维度）；将水库上游耕地通过生态补偿办法退出经营、引种芦苇恢复湿地40公顷——对应"退耕还湿"（修复维度）；H省秸秆还田与条耕轮耕保护黑土地——对应"保护性耕作"（修复维度）；"健康工作河流"理念既保护生态又承认适度开发——对应"平衡保护与开发"（预防维度）
- **结论**: 生态类对策按"补偿→修复→预防"三个层次组织
- **结果**: 规范答案覆盖：生态补偿、退耕还湿、保护性耕作、无害化处理、平衡保护与开发、保护性开发

## A2 — 触发场景 (Future Trigger)

### 用户会在什么情境下需要这个 skill?

1. 用户答对策题总是漏要点——写了3个规范词但标准答案是5-6个，维度不完整
2. 用户问"这类题该从哪几个角度答"——不知道怎么组织对策答案的框架结构
3. 用户需要检查答案完整性——写完答案后想确认有没有遗漏关键维度
4. 用户在练习"提出对策/建议/措施"类题目时，想确保覆盖所有得分维度

### 语言信号 (用户的话里出现这些就应激活)

- "提对策/建议/措施总是漏要点"
- "这类题该从哪几个角度答"
- "帮我检查答案有没有遗漏维度"
- "人才题/生态题/产业题的答题框架是什么"
- "不遗漏、维度、答题框架"

### 与相邻 skill 的区分

- 与 `sanmoshi-fanyi` 的区别: sanmoshi-fanyi 解决"怎么翻译"（口语→规范词的路径选择），本 skill 解决"怎么不遗漏"（确保覆盖所有维度）。先用翻译选词，再用模板检查完整性。
- 与 `kua-lingyu-zuhe` 的区别: kua-lingyu-zuhe 解决"跨领域组合"（多领域材料如何组织答案），本 skill 解决"单领域内不遗漏"（一个领域的维度结构）。本 skill 是 kua-lingyu-zuhe 的子模块——组合时每个领域内部的维度检查靠本 skill。
- 与 `wenti-celue-yinghe`（问题-对策映射与近义区分）的区别: wenti-celue-yinghe 解决"选对规范词"（问题↔对策的精准匹配），本 skill 解决"选全规范词"（维度覆盖不遗漏）。

## E — 可执行步骤 (Execution)

当 skill 被激活后, agent 应按以下步骤执行:

1. **判断材料涉及哪个政策领域**
   - 扫描材料关键词，确定属于8大领域中的哪一个或多个
   - 完成标准: 能明确说出"材料涉及人才领域""材料涉及生态+产业交叉"

2. **选取对应领域的维度模板**
   - 人才 → 引-育-留三维
   - 生态 → 补偿-修复-预防三段
   - 产业 → 标准-渠道-品牌三步
   - 民生就业 → 多元-就近-匹配三维
   - 普法 → 通俗-创新-丰富三维
   - 完成标准: 已选定1个或多个维度模板

3. **逐维度检查材料中是否有对应内容**
   - 带着模板的每个维度去扫材料，寻找该维度对应的内容线索
   - 完成标准: 每个维度都已标记"有/无"对应内容

4. **有 → 用对应规范词概括**
   - 找到对应内容后，调用 sanmoshi-fanyi 或 xinhao-ci-fanyi 翻译为规范词
   - 没有对应内容的维度 → 跳过，不硬凑
   - 完成标准: 每个有内容的维度都已翻译为规范词

5. **多领域 → 组合多个模板**
   - 如果材料涉及多个领域，重复步骤2-4，每个领域分别使用对应模板
   - 完成标准: 所有涉及领域的维度都已检查

6. **最终检查：每个维度是否都有答案**
   - 对照模板逐项核对，列出已覆盖和未覆盖的维度
   - 已覆盖的维度 = 答案要点；未覆盖的维度 = 确认材料确实没有涉及
   - 完成标准: 输出一份"维度覆盖清单"，确认无遗漏

## B — 边界 (Boundary)

### 不要在以下情况使用此 skill

- 题目问"存在什么问题/原因"——模板是对策用的，概括问题用 sanmoshi-fanyi
- 材料明显只涉及一个很窄的点（如只谈普法方式创新）——不需要强行套用完整三维模板
- 用户需要跨领域组合答案时——应配合 kua-lingyu-zuhe 使用，本 skill 只负责单领域内的维度检查

### 作者在书中警告的失败模式

- **机械套用模板而忽略材料实际内容**（对应 ce16: 规范词选错了比不用更糟糕）：模板是检查工具不是填空工具。如果材料只涉及"引才"和"留才"两个维度，就不要硬凑"育才"维度来凑数。模板的作用是"检查有没有遗漏"，不是"没有也要编出来"。
- **死记模板但不会识别材料中的触发信号**（对应 ce12: 死记硬背规范词但不会识别触发场景）：背住了"引-育-留"三维，但看到材料中"取消博士年龄限制"时无法识别这属于"引才"维度，模板就失去了意义。
- **把模板当作答案的唯一结构**（对应 ce17: 只教翻译不教思维）：模板帮助检查维度覆盖，但答案的组织还需要结合材料逻辑、题目要求和字数限制来灵活调整。

### 容易混淆的邻近方法论

- 本 skill vs 万能"多角度分析": 万能框架是"泛泛的多角度"，本 skill 是"每个领域有具体的预制维度"。"引-育-留"比"多方面考虑人才问题"具体得多，可直接套用和检查。
- 本 skill vs wenti-duice-yshe: wenti-duice-yshe 确保"选的词对"，本 skill 确保"选的词全"。两者互补：先选对，再选全。

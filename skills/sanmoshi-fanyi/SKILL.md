# 三模式翻译框架

## R — 原文 (Reading)

> 三种翻译模式：(1) 现象→术语："村民各过各的"→"发展集体经济"；(2) 做法→概括："给农作物上保险"→"提供农业保险服务"；(3) 问题→定性："卖的东西都一模一样"→"同质化"。翻译方向固定为：具体→抽象，口语→政策术语。规范表达须同时满足"准确"和"精炼"两个标准。
>
> — 《申论规范词积累课程》，全书贯穿（f01, p02, p12, p13）

---

## I — 方法论骨架 (Interpretation)

"三模式翻译框架"是申论答题中最底层的方法论：把材料中的"大白话"翻译成"政策规范表达"。

它不是一个单一操作，而是**三条翻译路径**，路径的选择取决于材料原文的表达类型：

1. **现象→术语**：材料描述一个具体现象或故事（"村民各过各的""直播间留不住客人"），需要提炼出这种现象在政策语言中叫什么（"发展集体经济""用户粘性不足"）。核心操作是**识别现象背后的政策本质**。

2. **做法→概括**：材料描述具体的操作动作（"给农作物上保险""城市学校派老师到农村手把手指导"），需要概括为抽象的对策表达（"提供农业保险服务""校际结对帮扶"）。核心操作是**提取动作的功能属性，忽略具体细节**。

3. **问题→定性**：材料描述负面现象（"卖的东西都一模一样""教到哪算哪"），需要用问题定性词概括（"同质化""课程缺乏系统性"）。核心操作是**判断问题的类别属性，而非复述症状**。

**关键判断**：先判断材料原文属于哪种表达类型（描述现象？叙述做法？暴露问题？），再选择对应的翻译路径。这个"先分类、再翻译"的排序是本框架区别于朴素翻译的核心——不是所有口语都需要同一种处理方式。

翻译的质量标准是两个维度的交集：**准确性**（不失真，核心含义保留）和**精炼性**（够简洁，用术语而非句子）。

## A1 — 书中的应用 (Past Application)

### 案例 1: 风林村"村寨银行"（现象→术语）
- **问题**: 材料描述"以前村民各过各的，谁也不管谁，现在有了村寨银行项目，养蜂种豆，通过专业合作社把农产品销往全国"
- **方法论的使用**: 材料在描述一个**现象/变化**——从分散到合作。识别现象背后的政策本质：合作社统购统销=集体经济的实现形式
- **结论**: "村民各过各的→合作社统销" 翻译为"发展集体经济"
- **结果**: 精准命中政策术语，"专业合作社"是"集体经济"的触发信号

### 案例 2: 北营村农作物保险（做法→概括）
- **问题**: 材料描述"2016年开始村里就提倡大家为农作物上保险，一亩地一年只需要交10.4元的保费，大旱后保险公司赔偿了1200多元"
- **方法论的使用**: 材料在叙述一个**具体做法**——买保险。提取功能属性而非细节：忽略金额（10.4元/亩）、忽略个案（杨大叔），提取动作的功能=风险保障机制
- **结论**: "给农作物上保险" 翻译为"提供农业保险服务"
- **结果**: 从具体操作升维为制度性对策表达，去除了所有细节数字

### 案例 3: 景区纪念品雷同（问题→定性）
- **问题**: 材料描述"卖的东西倒是不少，可和我们在别的景区看到的都是一样的，连纪念品都是一模一样的"
- **方法论的使用**: 材料在暴露一个**负面现象**——旅游商品雷同。判断问题的类别属性：不是"商品质量差"也不是"种类少"，而是"与别处趋同"
- **结论**: "卖的东西都一模一样" 翻译为"同质化"
- **结果**: 一个三字术语精准概括了整段描述的核心问题定性

## A2 — 触发场景 (Future Trigger) ★

### 用户会在什么情境下需要这个 skill?

1. 用户给出一段申论材料原文，要求将其翻译/概括为规范表达
2. 用户在练习申论小题时，不确定某段口语化材料应该用什么术语来概括
3. 用户想知道"这个具体描述在政策语言中叫什么"——从具体经验到制度命名的跳跃
4. 用户在翻译材料时纠结用哪个规范词，需要先判断材料的表达类型再选路径
5. 用户给出一个口语化的做法描述，需要概括为抽象的对策表达

### 语言信号 (用户的话里出现这些就应激活)

- "这段材料怎么翻译成规范表达"
- "口语转书面/口语转规范"
- "这个材料用什么术语概括"
- "帮我把这段话变规范"
- "材料变术语""大白话怎么改成官方话"

### 与相邻 skill 的区分

- 与 `glossary`（术语词典）的区别: glossary 用于"已有规范词，确认含义和用法"；本 skill 用于"还没有规范词，需要从材料翻译出规范词"。一个是查字典，一个是造翻译。
- 与 `kua-lingyu-zuhe`（跨领域组合）的区别: 本 skill 解决"单条材料→单个规范词"的翻译问题；跨领域组合解决"多个规范词→组织成完整答案"的排列问题。先翻译，再组合。
- 与 `xinhao-chufa`（信号词触发链）的区别: 信号词触发是本 skill 的加速版——当材料中出现明确关键词时可直接匹配，无需走完整翻译流程。本 skill 处理的是"没有明显信号词、需要从头推理"的场景。
- 与 `wenti-duice`（问题-对策映射）的区别: 问题-对策映射用于"已定性问题→匹配对策词"；本 skill 用于"原始材料→定性/概括/命名"。本 skill 是上游，问题-对策映射是下游。

## E — 可执行步骤 (Execution)

当 skill 被激活后，agent 应按以下步骤执行:

1. **判断材料原文的表达类型**
   - 逐句分析材料原文，判断其核心表达属于以下哪类：
     - **现象描述型**: 材料在讲述一个故事、描述一个变化、呈现一种状态（如"村民各过各的""直播间留不住人"）
     - **做法叙述型**: 材料在描述某个主体做了什么具体动作（如"给农作物上保险""一对一结对子"）
     - **问题暴露型**: 材料在揭示负面现象、不足、缺陷（如"卖的东西都一模一样""教到哪算哪"）
   - 完成标准: 能明确说出"这段材料属于X型"，且能说清判断依据

2. **选择对应翻译模式并执行翻译**
   - 若为**现象描述型** → 使用"现象→术语"模式：问自己"这个现象在政策语言中叫什么？"，提取现象背后的政策本质
   - 若为**做法叙述型** → 使用"做法→概括"模式：问自己"这个动作的功能属性是什么？去掉具体细节后它在做什么？"，提取动作的制度化表达
   - 若为**问题暴露型** → 使用"问题→定性"模式：问自己"这个问题的类别属性是什么？"，用名词化短语概括问题本质
   - 完成标准: 产出一个规范表达（术语/概括/定性词），且该表达符合动宾结构（对策类）或名词化结构（问题类）

3. **补充检验：如果材料是"结果/效果"描述**
   - 若材料描述的是某个措施的结果而非措施本身（如"腰包越来越鼓""村子颜值飙升"），需使用**因果倒推**：问"什么对策能产生这个结果？"，反向推出对策规范词
   - 完成标准: 能从结果准确倒推至产生该结果的对策规范词

4. **双标准检验**
   - **准确性检验**: 翻译后的规范表达是否保留了材料的核心含义？是否丢失了关键信息？
   - **精炼性检验**: 翻译后的表达是否足够简洁？是否仍然冗长或包含不必要的细节（金额、人名、具体数字）？
   - 完成标准: 翻译结果同时通过准确性和精炼性两项检验。如有冲突，优先保证准确性。

## B — 边界 (Boundary) ★

### 不要在以下情况使用此 skill

- 用户已经给出规范词，只需确认其含义或使用场景 → 应使用 glossary（术语词典）
- 用户已经翻译出多个规范词，需要组织成完整答案结构 → 应使用 kua-lingyu-zuhe（跨领域组合）
- 材料中出现明确的信号关键词（如"专业合作社""自治委员会"），可直接匹配 → 应使用 xinhao-ci-fanyi（信号词触发链）更高效
- 用户问的是"这个规范词和那个有什么区别" → 应使用 wenti-celue-yinghe（问题-对策映射与近义区分）中的近义区分规则

### 作者在书中警告的失败模式

- **抄原文陷阱** (ce01): 直接将材料原文搬进答案，没有经过翻译。材料原文是"叙事性语言"，答案要求"概括性语言"。抄原文=没有完成翻译任务=不得分。
- **俗语陷阱** (ce02): 把材料中生动的俗语或比喻（"教到哪算哪""东一榔头西一棒槌"）直接搬进答案。俗语是"症状描述"，规范词是"诊断结论"。
- **细节陷阱** (ce03): 把材料中的具体操作细节（金额、百分比、人名）写进答案。申论概括题要求的是"做了什么类型的事"，而非"具体怎么做的细节"。
- **泛化陷阱** (ce11): 用万能词（"环保不力""环境破坏"）替代精准的问题定性词（"环保错位"）。规范词的价值在于不仅说"有问题"，还说"问题的机制是什么"。
- **选错比不用更糟** (ce16): 强行套用背过的规范词但没有仔细审题，导致用在不合适的场景。宁可用自己的话准确描述，也不要硬套不合适的术语。

### 作者的盲点 / 时代局限

- 课程以"翻译"为核心，但翻译能力依赖于对中国公共政策话语体系的熟悉程度——如果用户不熟悉政策语境，仅靠翻译框架仍可能选错词
- 课程侧重于单个规范词的精准翻译，对"多个规范词之间的逻辑关系如何组织"着墨较少（由后续 skill 补充）

### 容易混淆的邻近方法论

- **具体→抽象升维** (f07): 这是本框架"做法→概括"模式的子机制，不是独立方法论。本框架已包含升维操作。
- **同类合并归纳法** (f11): 当材料并列列举多个具体事项时，需先合并再翻译。这是本框架的补充技巧，不是独立框架。
- **因果倒推翻译法** (f10): 当材料描述结果而非做法时使用。已整合进本框架的步骤3。
- **同义表达归一法** (f17): 不同材料表达可能指向同一规范词。这是本框架的识别能力补充。

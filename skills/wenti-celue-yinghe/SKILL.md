# 问题-对策映射与近义区分

## R — 原文 (Reading)

> 规范词选错了比不用更糟糕。每个规范词的应用场景定义中，隐含了"看到X类问题→使用Y类对策规范词"的对应关系。选用规范词时，进行双重校验：定义校验——材料的含义是否匹配该规范词的定义？场景校验——材料的具体情境是否出现在该规范词的典型应用场景中？只有两层都通过，才能确认选用该词。
>
> — 《申论规范词积累课程》，全书贯穿（民生福祉、基层治理模块）

---

## I — 方法论骨架 (Interpretation)

申论材料中的每一个问题描述，都暗含着对应的对策规范词——这不是随机的，而是政策逻辑本身的结构性映射。"看病难"不是笼统的问题，它背后有三条不同的解决路径：远程医疗、分级诊疗、优质资源下沉——选哪条路径，取决于材料描述的具体情境。

核心逻辑是：**问题决定对策方向，场景决定具体选词**。先识别材料中的问题性质（是资源不足、制度缺失、还是意识薄弱），再在该问题对应的对策规范词候选集中，通过"定义-场景双重校验"锁定最精准的一个。

近义区分之所以必要，是因为"选错比不用更糟"。阅卷人看到明显不匹配的规范词，会认为考生生搬硬套。例如把"分级诊疗"写成"远程医疗"，虽然都在解决看病难，但暴露了考生不理解两者的本质区别。

操作方法：**先对定义，再对场景**。定义校验解决"大方向对不对"，场景校验解决"细节匹不匹配"。两层都通过才能确认选用；任一层通不过就换词。最后一步是替换验证——把选定的词换成其他近义词，看是否失真。

## A1 — 书中的应用 (Past Application)

### 案例 1: 远程医疗 vs 分级诊疗 vs 优质资源下沉

- **问题**: 材料描述"看病难"问题，三个规范词都似乎可选
- **方法论的使用**: 定义校验——远程医疗=以技术为依托的跨地域医疗协作；分级诊疗=大医院与基层组建医联体、重症轻症分流转诊；优质资源下沉=三甲资源向基层倾斜。场景校验——材料说"三甲医院专家到乡镇卫生院坐诊"→ 主体是三甲专家、行为是向下流动 → 命中"优质资源下沉"；材料说"通过5G平台进行远程手术指导"→ 关键词是技术平台和跨地域 → 命中"远程医疗"；材料说"大医院看急重症，康复期转回社区"→ 关键词是分流和转诊 → 命中"分级诊疗"
- **结论**: 三者共同目标是解决"看病难"，但路径完全不同——技术路径、制度路径、资源路径
- **结果**: 精准区分避免"看到医疗就选一个"的粗放匹配

### 案例 2: 政务服务下沉 vs 一站式服务

- **问题**: 材料描述群众办事更方便了，两个规范词都像正确答案
- **方法论的使用**: 定义校验——政务服务下沉=行政服务向基层/农村延伸（空间下沉）；一站式服务=多个事项集中到一个平台/窗口办理（事项集中）。场景校验——材料说"办临时身份证不用再去政务中心，辖区派出所就能办"→ 强调服务地点下移到基层 → 命中"政务服务下沉"；材料说"一网通办、一个窗口搞定所有审批"→ 强调事项集中办理 → 命中"一站式服务"
- **结论**: 前者是"往下移"，后者是"往一处聚"，方向不同
- **结果**: 区分了"空间下沉"和"事项集中"两种不同的便民服务路径

### 案例 3: 医养结合 vs 智慧养老

- **问题**: 材料描述养老服务改善，两个规范词都可能匹配
- **方法论的使用**: 定义校验——医养结合=在生活照料基础上提供医疗卫生服务（医疗嵌入）；智慧养老=利用互联网、物联网、大数据等信息技术提供便捷养老服务（技术嵌入）。场景校验——材料说"社区为老人建立健康档案、配备全科医生定期巡诊"→ 核心是医疗服务嵌入养老 → 命中"医养结合"；材料说"一键呼叫设备、智能手环实时监测心率"→ 核心是智能设备赋能 → 命中"智慧养老"
- **结论**: 同为养老改善，一个靠"医疗功能"，一个靠"信息技术"
- **结果**: 如果材料同时涉及医疗和智能设备，两个规范词可同时使用

## A2 — 触发场景 (Future Trigger) ★

### 用户会在什么情境下需要这个 skill?

1. 用户在多个看似都对的规范词中犹豫不决，问"该用哪个词"时
2. 用户直接问"这个词和那个词有什么区别"，需要精准近义区分时
3. 用户从材料中提取了问题描述，需要推导出对应的对策规范词时
4. 用户写出的答案用了规范词但感觉"不太对"，需要校验和替换时

### 语言信号 (用户的话里出现这些就应激活)

- "选哪个词""用哪个更准确"
- "这两个词有什么区别"
- "这个问题对应什么对策"
- "用词精准""近义区分""选词"

### 与相邻 skill 的区分

- 与 `sanmoshi-fanyi`（三模式翻译）的区别: 翻译解决"如何从口语到规范词"的路径问题，本skill解决"多个候选规范词中如何选最精准的"的判断问题
- 与 `kua-lingyu-zuhe`（跨领域组合）的区别: 组合解决"答案怎么排队、怎么跨领域搭配"的组织问题，本skill解决"单个词的精准选择"问题
- 与 `xinhao-ci-fanyi`（信号词触发链）的区别: 信号词解决"快速识别"的提速问题，本skill解决"识别后需要精确区分"的校验问题

## E — 可执行步骤 (Execution)

当 skill 被激活后，agent 应按以下步骤执行:

1. **判断题目要求方向**
   - 确认题目问的是"问题""对策"还是"特点/经验"
   - 完成标准: 明确了答案应使用"问题描述类""对策表达类"还是"特征概括类"规范词
   - 判停条件: 若题目方向已明确，直接进入步骤2

2. **从材料中提取问题描述**
   - 定位材料中的关键问题表述（负面现象、困难描述、不足指出）
   - 用自然语言概括材料的核心问题是什么
   - 完成标准: 能用一句话描述"材料反映的核心问题"

3. **在规范词库中匹配候选对策词**
   - 根据问题性质（资源不足/制度缺失/意识薄弱/技术落后/管理混乱等），在对应领域的规范词中检索候选词
   - 列出所有可能匹配的规范词（通常2-4个）
   - 完成标准: 得到一份候选规范词清单

4. **多重候选 → 定义校验 + 场景校验 → 选出最精准的一个**
   - 定义校验: 逐一核对每个候选词的定义，看材料含义是否匹配
   - 场景校验: 核对每个候选词的典型应用场景，看材料情境是否命中
   - 两层都通过的词 → 确认为最终选择
   - 若仍有多个通过 → 进入步骤5
   - 完成标准: 从候选集中锁定1个最优词

5. **验证：替换测试**
   - 将选定的词替换为其他近义词，看替换后是否失真
   - 如果替换后意思明显改变 → 说明选词精准，通过
   - 如果替换后意思差不多 → 说明区分不够，需重新回到步骤4
   - 完成标准: 确认选定的词与近义词之间存在不可替代的差异

## B — 边界 (Boundary) ★

### 不要在以下情况使用此 skill

- 材料没有明确的问题描述，只是中性叙述时 — 没有"问题"就无法启动"问题→对策"映射链
- 题目只要求抄写原文或简单概括时 — 不涉及规范词选择
- 只需要翻译而不需要在多个近义词中做选择时 — 这是 sanmoshi-fanyi 的工作

### 作者在书中警告的失败模式

- **近义混淆**: 看到"看病难"就笼统写"医疗改革"，不区分远程医疗、分级诊疗、优质资源下沉的具体路径差异（ce04-ce05, p20）
- **三种"下沉"混淆**: 看到"向下"就笼统写"下沉"，不区分干部下沉（人员流动）、政务服务下沉（事权下放）、优质资源下沉（资源倾斜）的主体和内涵差异（ce05）
- **选错比不用更糟**: 为了"显得专业"强行套用不匹配的规范词，结果暴露了不理解材料（ce16）
- **问题-对策倒置**: 题目问对策却答了问题，题目问问题却答了对策（ce13, ce30）

### 容易混淆的邻近方法论

- 与 `sanmoshi-fanyi` 容易混淆: 翻译skill解决"怎么翻译"，本skill解决"翻译后选哪个词更精准"
- 与 `xinhao-ci-fanyi` 容易混淆: 信号词skill是"条件反射式的快速匹配"，本skill是"需要仔细区分的慢思考"

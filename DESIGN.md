# DESIGN.md · 考编备考全攻略生成器

## 产品灵魂 · 意象锚点
**清晨的备考自习室**：靠窗的木质长桌、一本翻开的错题本、旁边一枚素色印章、窗外淡淡的米黄晨光。产品要传达的不是"科技感 SaaS 工具"，而是"一位替你把节奏理清楚的学长学姐"。信息密度高、语气克制、有秩序感。

严格回避的第一直觉：科技蓝渐变、紫粉赛博风、荧光高饱和、深色炫酷仪表盘 —— 与"考编"这一严肃、体制内、长期备考场景完全不匹配。

## 视觉策略
- 排版基调：档案纸/公文纸的秩序感，卡片 + 分组 + 长表格，信息层级依靠字号、字重、极细分隔线（.5px）与留白，而不是靠色块堆叠。
- 单页应用（Single HTML, No Build Step），左侧固定表单面板 + 右侧多 Tab 内容区，桌面双栏、移动端单栏。

## Design Tokens

### 色彩
- 主色 `--P: #F3A04C`（朱砂橘）：来自公章印泥的暖橘，只用于主按钮、Logo 印章、进度条渐变，占比 ≤ 10%。
- 主色深 `--A: #E58522`：hover / 强调文字。
- 背景 `--BG: #F2F2F4`（档案纸浅灰）；卡片 `--CA: #FFFFFF`；面板细底 `--PA: #FEFDFC`。
- 状态：`--OK: #40B43E`（充裕，墨绿呼应稳重）／ `--WA: #EBA400`（注意）／ `--ER: #E65C53`（紧张）。
- 文本三档：`--T1: rgba(0,0,0,.82)` / `--T2: rgba(0,0,0,.52)` / `--T3: rgba(0,0,0,.36)`。
- 描边采用 `.5px solid rgba(0,0,0,.09)`，禁止使用 1px 粗描边 + 高饱和颜色（会显得廉价）。

### 字体
- 优先系统栈：`-apple-system, BlinkMacSystemFont, "PingFang SC", "Hiragino Sans GB", "Microsoft YaHei", sans-serif`。产品文档硬性要求"独立 HTML，无外部依赖"，禁止引入 Google Fonts / 网络字体。
- 字号：正文 12–13px，卡片标题 14px，Hero 主标题 17px，KPI 数字 22px。中文行高 1.5–1.7，避免拥挤。

### 间距 / 圆角 / 阴影
- 间距节奏：8 / 12 / 14 / 16 / 20（px），避免出现 5、7、11 之类"乱数"。
- 圆角：控件 8px，卡片 10–12px，标签胶囊 999px（用于状态 Chip）。
- 阴影：不使用彩色发光/大范围投影，仅用极浅细分隔线勾勒卡片轮廓。

### 动效
- 全站仅保留控件 hover 的 `background/border-color .12–.15s` 过渡，禁止入场动画、粒子、渐变漂移。备考场景强调专注，动效越少越好。

## 布局与响应式
- 桌面（≥ 820px）：左栏 308px 固定 + 右栏自适应；左栏 sticky。
- 移动（< 820px）：单列堆叠，KPI 从 4 列降为 2 列，matrix 折叠为单列。
- 打印：隐藏左栏 / Tabs / 按钮，右栏白底铺满，所有 Tab 内容同时展示，方便打印/保存 PDF。

## 组件规范
- 按钮：主 `.btn`（朱砂橘）+ 次 `.btn2`（浅灰底），高度 34px，不使用大号 CTA。
- Chip 标签：胶囊形，色板取 `--OK / --WA / --ER` 各 14% 混透明作背景，边框同色 26%。
- 表格：细分隔线 + 表头灰字，禁止斑马纹。
- 进度条：轨道 8px 圆角，填充采用主色 → 主色深线性渐变。

## 交互与状态
- 所有表单控件 `input / change` 立即触发全量渲染，无需"生成"按钮（"生成报告"按钮仅切换到报告 Tab）。
- 倒排计划的天数颜色规则：> 120 天绿、60–120 橙、< 60 红、< 0 灰。

## 内容与文案
- 简体中文，语气务实、克制，不使用感叹号堆叠、不出现"YYDS/上岸稳了"等口语。
- 数字与百分比保持半角，中文标点使用全角。

## 设计禁忌
- 不加深色模式（备考白天场景为主）。
- 不加任何图表库、动画库、字体 CDN。
- 不使用彩色渐变按钮、玻璃拟态、Neumorphism。
- 不使用 Emoji 作为唯一信息载体（作为分类前缀 OK，但重要文字仍需中文）。
